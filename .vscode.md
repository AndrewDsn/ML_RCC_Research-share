## Python Version & Virtual Environment

### Check Python Version

```bash
python3.12 --version
# Expected output: Python 3.12.x

# If python3.12 is not available, check alternatives:
python3 --version  # Alternative
python --version   # Alternative
```

### Create & Activate Virtual Environment with Python 3.12

```bash
cd /workspaces/ML_RCC_Research-share/project

# Create virtual environment with Python 3.12
python3.12 -m venv .venv
# Or on systems where python3.12 is aliased as python:
python -m venv .venv

# Activate on Linux/macOS
source .venv/bin/activate

# Activate on Windows PowerShell
.venv\Scripts\Activate.ps1

# Activate on Windows Command Prompt
.venv\Scripts\activate.bat
```

```bash
pip install --upgrade pip setuptools wheel
```

## System Dependencies

### Linux (Ubuntu/Debian)

Install build tools and scientific libraries required for OpenSeesPy:

```bash
# Update package manager
sudo apt update

# Install build essentials
sudo apt install -y build-essential python3-dev

# Install Fortran compiler and BLAS/LAPACK (required for OpenSeesPy)
sudo apt install -y gfortran libblas-dev liblapack-dev libopenblas-dev

# Install HDF5 libraries (for h5py)
sudo apt install -y libhdf5-dev

# Install additional scientific libraries
sudo apt install -y libatlas-base-dev
```

### macOS

```bash
# Using Homebrew
brew install gcc gfortran openblas lapack hdf5
```

### Windows

Use pre-built wheels or install via Anaconda/Miniconda:

```bash
# Via conda (recommended for Windows and best for Python 3.12)
conda create -n seismic python=3.12
conda activate seismic
conda install -c conda-forge openseespy numpy scipy pandas
pip install -r requirements.txt
```

## Quick Start Setup Script

### All-in-One Setup (Linux/macOS)

Save as `setup.sh` and run: `bash setup.sh`

```bash
#!/bin/bash

set -e  # Exit on error

echo "🚀 ML Seismic Drift Research - Setup Script"
echo "==========================================="
echo "Python Version: 3.12"
echo ""

# Check Python 3.12 version
echo "✓ Checking Python 3.12 version..."
python3.12 --version || { echo "Python 3.12 not found! Please install Python 3.12 first."; exit 1; }

# Navigate to project
cd /workspaces/ML_RCC_Research-share/project

# Create virtual environment with Python 3.12
echo "✓ Creating virtual environment with Python 3.12..."
python3.12 -m venv .venv

# Activate virtual environment
echo "✓ Activating virtual environment..."
source .venv/bin/activate

# Upgrade pip
echo "✓ Upgrading pip, setuptools, and wheel..."
pip install --upgrade pip setuptools wheel

# Install all requirements
echo "✓ Installing Python packages (this may take 5-10 minutes)..."
pip install -r requirements.txt

# Verify installation
echo "✓ Verifying critical imports..."
python3.12 << 'EOF'
import sys
try:
    import numpy
    import pandas
    import scipy
    import sklearn
    import openseespy.opensees as ops
    import tensorflow as tf
    import xgboost
    import shap
    print("✅ All critical packages imported successfully!")
    print(f"Python: {sys.version}")
    print(f"NumPy: {numpy.__version__}")
    print(f"Pandas: {pandas.__version__}")
    print(f"OpenSeesPy: installed")
    print(f"TensorFlow: {tf.__version__}")
    print(f"XGBoost: {xgboost.__version__}")
except ImportError as e:
    print(f"❌ Import error: {e}")
    sys.exit(1)
EOF

echo ""
echo "✅ Setup Complete!"
echo "==========================================="
echo "Next steps:"
echo "  1. Activate environment: source .venv/bin/activate"
echo "  2. Start Jupyter: jupyter lab --notebook-dir=notebooks"
echo "  3. Run tests: pytest tests/ -v"
echo ""
```

## Troubleshooting

### OpenSeesPy Installation Issues

**Error:** `ModuleNotFoundError: No module named 'opensees'`

**Solutions:**
```bash
# 1. Ensure build tools are installed
sudo apt install build-essential gfortran libblas-dev liblapack-dev

# 2. Reinstall openseespy
pip uninstall openseespy -y
pip install openseespy --no-cache-dir
```

### TensorFlow/CUDA Issues

**Error:** `Could not load dynamic library 'libcudart.so.11.0'`

**Solutions:**
```bash
# Install CPU-only version (recommended for development)
pip uninstall tensorflow -y
pip install tensorflow-cpu>=2.13.0

# Or use conda (handles CUDA automatically)
conda install tensorflow::tensorflow
```

### Jupyter Kernel Issues

```bash
# Install ipykernel for Jupyter
pip install ipykernel

# Add kernel to Jupyter
python -m ipykernel install --user --name seismic-drift --display-name "Python (Seismic)"
```

### Import Errors

```bash
# Clear pip cache and reinstall
pip cache purge
pip install -r requirements.txt --no-cache-dir --force-reinstall
```
